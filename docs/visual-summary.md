## 3. Prompt vs Context vs Harness 对比

```
┌──────────────────────────────────────────────────────────────────────┐
│  PROMPT ENGINEERING                                                  │
│  = 告诉实习生该做什么                                                 │
│                                                                      │
│  "请写一个Python函数，接收整数列表并返回总和。                        │
│   遵循PEP8规范，包含类型提示和文档字符串。"                           │
│                                                                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  焦点: 单次指令优化     工具: ChatGPT, Claude                         │
└──────────────────────────────────────────────────────────────────────┘

                           ⬇️ 扩展 ⬇️

┌──────────────────────────────────────────────────────────────────────┐
│  CONTEXT ENGINEERING                                                 │
│  = 给实习生完整的项目资料和历史记录                                    │
│                                                                      │
│  系统提示词 + 代码库文档 + 相关文件 + 用户偏好 + 实时数据 + 任务描述   │
│                                                                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  焦点: 信息环境设计     工具: RAG, Memory, MCP                        │
└──────────────────────────────────────────────────────────────────────┘

                           ⬇️ 扩展 ⬇️

┌──────────────────────────────────────────────────────────────────────┐
│  HARNESS ENGINEERING                                                 │
│  = 建立公司制度、检查流程和防错机制                                    │
│                                                                      │
│  架构约束 ┃ AGENTS.md ┃ 验证工具 ┃ 自动测试 ┃ 人机回环 ┃ 安全沙箱    │
│                                                                      │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━   │
│  焦点: 系统工程         原则: 让错误不可能发生                        │
│  座右铭: "发现错误 → 工程化修复 → 永不重蹈覆辙"                       │
└──────────────────────────────────────────────────────────────────────┘
```

## 4. 三种核心架构模式

### 模式1: 单 Agent 监督者
```
┌─────────┐     ┌─────────────────────────────────────────┐     ┌─────────┐
│  User   │────▶│  Single-Agent Supervisor Harness        │────▶│  Result │
│ Request │     │                                         │     │ Output  │
└─────────┘     │  ┌─────────┐    ┌─────┐    ┌────────┐  │     └─────────┘
                │  │  LLM    │◄───│Tools│◄───│ Memory │  │
                │  │ Loop    │    └─────┘    └────────┘  │
                │  └─────────┘                           │
                │                                         │
                │  适用: 有界任务（客服、问答）            │
                └─────────────────────────────────────────┘
```

### 模式2: 初始化器-执行器分离
```
PHASE 1: 一次性初始化
┌─────────────────────────────────────────┐
│  Initializer Agent                      │
│  - 创建项目结构                         │
│  - 生成功能清单 (features.json)         │
│  - Git 初始提交                         │
└──────────────────┬──────────────────────┘
                   │
                   ▼
         ┌───────────────────┐
         │ Project State     │
         │ (持久化环境)       │
         └─────────┬─────────┘
                   │
PHASE 2: 重复执行   │
                   │
┌──────────────────┴──────────────────────┐
│  Executor Session N                     │
│  1. 读取状态  2. 完成功能  3. 测试      │
│  4. 提交      5. 更新状态  6. 退出      │
└─────────────────────────────────────────┘

适用: 长期编码任务（如 Anthropic 的 claude.ai 克隆项目）
```

### 模式3: 多 Agent 协调
```
                    ┌─────────────────┐
                    │   Coordinator   │
                    │   (协调器)      │
                    └───────┬─────────┘
                            │
        ┌───────────────────┼───────────────────┐
        │                   │                   │
        ▼                   ▼                   ▼
   ┌──────────┐       ┌──────────┐       ┌──────────┐
   │Researcher│──────▶│ Writer   │──────▶│ Reviewer │
   │ (研究)   │       │ (写作)   │       │ (审核)   │
   └──────────┘       └──────────┘       └─────┬────┘
                                               │
                                               ▼
                                         ┌──────────┐
                                         │  Output  │
                                         └──────────┘

适用: 复杂项目（研究报告、内容创作、代码审查）
```

## 5. 为什么 LLM 需要 Harness?

```
┌──────────────────────────────────────────────────────────────────────┐
│                      LLM 原生限制                                     │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ❌ 无状态        每次新会话都是"盲人"，不知道之前发生了什么         │
│                                                                      │
│  ❌ 上下文有限    即使1M token也会填满，长上下文性能下降             │
│                                                                      │
│  ❌ 工具幻觉      可能调用不存在的API，参数类型错误                  │
│                                                                      │
│  ❌ 状态丢失      网络中断或重启后进度归零                           │
│                                                                      │
│  ❌ Context Rot   长上下文中重要信息被淹没（Lost in the Middle）     │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
                                ⬇️
                    ┌───────────────────┐
                    │   AGENT HARNESS   │
                    │     解决方案       │
                    └─────────┬─────────┘
                              │
        ┌─────────────────────┼─────────────────────┐
        │                     │                     │
        ▼                     ▼                     ▼
   ┌──────────┐         ┌──────────┐         ┌──────────┐
   │ 状态持久化 │         │ 上下文管理 │         │ 工具验证  │
   │ State     │         │ Context   │         │ Tool      │
   │ Persistence│        │ Management│         │ Validation│
   └──────────┘         └──────────┘         └──────────┘
```

## 6. Harness Engineering 工作流程

```
┌──────────────────────────────────────────────────────────────────────┐
│                    HARNESS ENGINEERING 循环                          │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   ┌───────────────────────────────────────────────────────────────┐ │
│   │  STEP 1: Agent 执行任务                                        │ │
│   └──────────────────────────────┬────────────────────────────────┘ │
│                                  │                                   │
│                                  ▼                                   │
│   ┌───────────────────────────────────────────────────────────────┐ │
│   │  STEP 2: 失败发生?                                             │ │
│   │         (错误/幻觉/未完成)                                      │ │
│   └──────────────────────────────┬────────────────────────────────┘ │
│                                  │                                   │
│                    ┌─────────────┴─────────────┐                    │
│                    │                             │                    │
│                    ▼                             ▼                    │
│         ┌─────────────────────┐    ┌─────────────────────┐           │
│         │   是 → 工程化修复    │    │   否 → 继续监控     │           │
│         └──────────┬──────────┘    └─────────────────────┘           │
│                    │                                                  │
│                    ▼                                                  │
│   ┌───────────────────────────────────────────────────────────────┐  │
│   │  STEP 3: 系统性修复 (Mitchell Hashimoto 原则)                  │  │
│   │                                                                │  │
│   │  ┌───────────────────────────────────────────────────────────┐│  │
│   │  │ 修复1: 更新 AGENTS.md                                      ││  │
│   │  │         添加规则防止此失败模式                              ││  │
│   │  └───────────────────────────────────────────────────────────┘│  │
│   │                                                                │  │
│   │  ┌───────────────────────────────────────────────────────────┐│  │
│   │  │ 修复2: 构建验证工具                                        ││  │
│   │  │         使正确行为可机械验证                                ││  │
│   │  └───────────────────────────────────────────────────────────┘│  │
│   │                                                                │  │
│   │  ┌───────────────────────────────────────────────────────────┐│  │
│   │  │ 修复3: 调整架构约束                                        ││  │
│   │  │         通过 linter/测试强制执行                            ││  │
│   │  └───────────────────────────────────────────────────────────┘│  │
│   └───────────────────────────────────────────────────────────────┘  │
│                                  │                                   │
│                                  ▼                                   │
│   ┌───────────────────────────────────────────────────────────────┐  │
│   │  STEP 4: Agent 再也不会犯同样错误 ✅                            │  │
│   └───────────────────────────────────────────────────────────────┘  │
│                                  │                                   │
│                                  └───────────────────────────────────┘
│                                                      │
│                                                      ▼
│                                           (回到 STEP 1)
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

## 8. 关键要点速记

```
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│   🔑 核心公式                                                        │
│                                                                      │
│   Agent = LLM Model + Harness                                        │
│           (大脑)      (身体+环境)                                     │
│                                                                      │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   🔑 Harness Engineering 座右铭                                      │
│                                                                      │
│   "发现 Agent 犯错 → 工程化环境使其物理上无法再次犯错"               │
│                                                                      │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   🔑 三层防御                                                        │
│                                                                      │
│   L1: 架构约束 → 强制代码结构                                        │
│   L2: AGENTS.md → 累积项目知识                                       │
│   L3: 工具验证 → 机械正确性检查                                      │
│                                                                      │
├──────────────────────────────────────────────────────────────────────┤
│                                                                      │
│   🔑 关键区别                                                        │
│                                                                      │
│   Prompt: 说什么      (在窗口内)                                     │
│   Context: 给什么     (填充窗口)                                     │
│   Harness: 在什么环境运行 (整个系统)                                 │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```
